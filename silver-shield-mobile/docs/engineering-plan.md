# 工程完善方案

## 埋点方案

建议最小事件集：

- 登录成功 / 失败
- 角色切换
- 打开聊天页
- 发送消息
- 触发风险识别
- 点击风险详情
- 发起求助
- 发起通话 / 接听 / 挂断

字段建议：

- `userRole`
- `page`
- `contactId` / `elderId`
- `riskLevel`
- `networkStatus`
- `appEnv`

## 日志采集方案

- 前端记录页面异常、接口失败、风险识别失败、求助失败、通话状态异常
- 日志分级：`info`、`warn`、`error`
- 对外发送前统一脱敏手机号、链接、消息文本中的身份证/银行卡片段

## 崩溃监控方案

- H5 阶段先接前端异常采集
- 小程序阶段补充运行时错误、Promise 未处理异常、资源加载失败
- App 化阶段再接原生崩溃监控

## 版本发布流程

1. 开发分支自测
2. H5 演示环境验收
3. 微信小程序专项验证
4. 生成演示包和测试包
5. 记录版本号、变更项和回滚方案

## 测试包与演示包管理

- 演示包：固定使用 `demo` 环境，不依赖真实后端
- 测试包：连接测试环境，验证真实接口
- 发布说明中必须标明接口地址、账号、测试范围和限制项

## 环境区分

当前已落地：

- `development`
- `demo`
- `production`

对应脚本：

- `npm run dev:h5`
- `npm run dev:h5:demo`
- `npm run build:h5`
- `npm run build:h5:demo`
