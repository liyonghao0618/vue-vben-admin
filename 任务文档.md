# 守护桑榆——反电诈 AI 助手
## 任务文档

**版本**：V1.0  
**文档目标**：将需求文档与技术选型进一步落地为可执行的研发任务清单，用于项目排期、模块拆分、人员分工与阶段验收。

## 1. 项目任务目标

基于现有 `vue-vben-admin` 前端工程，建设一套面向老年用户、子女用户、社区工作人员和系统管理员的反诈协同平台，形成“识别风险、及时提醒、联动通知、社区处置、后台配置”的业务闭环。

V1.0 的核心任务不是一次性完成全部 AI 能力，而是优先交付一个可演示、可上线、可持续迭代的最小可行版本。

## 2. 本期建设范围

### 2.1 纳入 V1.0 的范围

- 账号登录、角色区分、基础权限控制
- 老年端基础页面与适老化交互
- 子女端风险通知与监护面板
- 社区端辖区总览、重点人员管理、工单处置
- 管理端用户、角色、规则、内容、系统配置
- 短信风险识别
- 通话文本风险识别基础版
- 风险分级、原因说明、建议动作输出
- 高风险事件通知子女与社区
- 风险规则、风险词库、提示模板后台配置

### 2.2 暂不纳入 V1.0 的范围

- 高精度语音模型训练
- 方言识别增强
- 复杂多模态识别
- 短信外呼、自动语音回访等重型外部能力
- 复杂消息中间件集群化建设

## 3. 总体任务拆解

### 3.1 前端任务

基于 `vue-vben-admin` 实现多角色业务页面，并完成适老化界面设计与核心交互联通。

**主要任务**

- 搭建项目菜单、路由、权限映射
- 建设老年端首页、风险提醒页、一键求助页、亲属绑定页、防骗知识页、设置页
- 建设子女端首页总览、老人列表页、风险详情页、通知记录页、监护设置页
- 建设社区端数据看板、辖区老人列表、风险工单页、宣教管理页、统计报表页
- 建设管理端用户管理、角色权限、风险规则、内容管理、系统配置页面
- 封装告警通知、风险标签、趋势图表、处置记录等通用组件
- 对老年端实现大字号、大按钮、高对比度、少步骤交互

**交付结果**

- 前端页面原型落地
- 基础 API 联调完成
- 多角色可登录并进入对应页面
- 2026-04-14：老年端风险提醒页已完成第一版业务实现，支持风险等级、原因说明、建议动作和一键求助跳转。
- 2026-04-14：3.1 前端核心闭环页面已全部补齐，可演示老年端求助与亲属绑定、子女端监护配置、社区端重点对象与工单处置、管理端角色/规则/内容/系统配置展示。

### 3.2 后端任务

基于 `FastAPI` 完成业务 API、权限认证、核心数据模型与通知协同能力。

**主要任务**

- 搭建后端基础工程结构
- 建立用户、角色、绑定关系、告警、通知、工单、内容、规则等核心数据模型
- 实现登录认证、JWT 鉴权、角色权限控制
- 实现用户管理、亲属绑定、告警查询、工单流转、内容管理、规则配置等 API
- 实现高风险事件通知接口
- 提供 WebSocket 或轮询所需的实时消息接口
- 补充审计日志与关键操作留痕

**交付结果**

- 后端接口文档
- 可运行的业务服务
- 基础权限与数据流转闭环

### 3.3 风险识别任务

建设独立的风险识别模块，优先采用规则引擎方案完成短信和通话文本的风险识别。

**主要任务**

- 设计风险词库、规则库、命中策略
- 建设短信文本检测能力
- 建设通话转写文本检测基础能力
- 输出风险等级、命中原因、建议动作
- 支持规则后台配置与版本更新
- 预留后续模型服务接入接口

**交付结果**

- 短信识别基础服务
- 通话文本识别基础服务
- 风险分析结果结构化输出

### 3.4 数据与基础设施任务

围绕业务数据存储、缓存、通知和部署能力完成基础支撑建设。

**主要任务**

- 建立 `PostgreSQL` 数据库表结构
- 建立 `Redis` 缓存、验证码、通知队列、热点配置存储
- 配置数据库迁移能力
- 设计日志、监控、异常告警基础方案
- 配置开发、测试、生产环境的基础部署参数

**交付结果**

- 数据库初始化脚本与迁移脚本
- 缓存与通知基础能力可用
- 项目基础部署方案明确

## 5. 推荐研发阶段

### 第一阶段：项目基础搭建

- 明确页面信息架构与接口边界
- 初始化前后端目录结构
- 搭建用户、角色、菜单、认证基础能力
- 完成数据库设计与迁移机制

**阶段目标**

系统具备基础登录、权限控制和开发运行环境。

### 第二阶段：核心业务闭环建设

- 完成老年端、子女端、社区端、管理端核心页面
- 完成亲属绑定、风险提醒、通知、工单、内容管理
- 完成短信识别与通话文本识别基础版
- 打通告警生成到通知分发的主链路

**阶段目标**

完成“风险识别 -> 风险提醒 -> 子女通知 -> 社区处置 -> 后台配置”的核心闭环。

### 第三阶段：优化与验收

- 优化适老化体验
- 优化告警展示、统计分析、处置记录
- 补充日志、权限校验、异常处理
- 开展联调、测试、验收与演示准备

**阶段目标**

系统达到演示与初步上线条件。

## 8. 风险与注意事项

- 风险识别结果只能作为辅助判断，不能直接替代人工决策。
- 老年端设计必须优先考虑可读性和操作简洁性。
- 涉及个人信息、通话、短信内容时必须做好授权与合规提示。
- 第三方能力不可用时，应保证基础规则提醒仍能运行。
- V1 不宜过早引入复杂模型和重型架构，避免影响交付节奏。
